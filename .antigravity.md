# Règles et Configuration Spécifiques pour Antigravity (Agents IA)

Afin d'éviter tout système non contrôlé ou chaotique, la participation des agents d'intelligence artificielle (notamment Antigravity) à ce projet est régie par ce fichier de règles qui ne peut en aucun cas être contourné.

## 1. Respect du Périmètre d'Intervention (Scope)

- **Limitation stricte** : L'IA ne doit formuler, générer ou appliquer aucune modification qui serait explicitement hors du périmètre demandé par l'utilisateur lors du prompt de départ ou des instructions courantes.
- **Justification de changement structurel** : Toute modification en profondeur de l'architecture existante, des modèles de données, des routes principales, ou l'ajout de dépendances lourdes (packages NPM/Composer) ne peut pas se faire silencieusement. L'IA doit explicitement justifier la raison de ces changements aux yeux de l'humain avant implémentation (notamment sous la forme d'un Implementation Plan).

## 2. Processus de Validation Exigé (Quality Gates)

L'IA n'est pas autorisée à déclarer un travail "terminé" si les étapes suivantes n'ont pas été prouvées ou validées :
- **Qualité de code (Lint / PSR / Standard)** : Le code JS/PHP écrit doit respecter les règles du projet sans introduire de hacks ou de régressions dans l'existant.
- **Sécurité et Sécurisation des Accès** : Avant toute mise en place d'une fonctionnalité ou d'une route, l'IA se doit d'assurer et rappeler les nécessités sur les middlewares de sécurité (ex: protections XSRF, Auth, vérifications RBAC/rôles).
- **Tests** : Il est nécessaire de vérifier, soit via les tests du projet, soit via une analyse poussée des répercussions, que rien n'a été détruit en cours de manipulation. De plus, il faut produire les tests associés aux nouvelles pages ou logiques métiers critiques selon la stack existante (PHPUnit, Cypress, etc.).
- **Mise à jour de la documentation** : Si l'IA modifie le code, elle a l'obligation de mettre à jour la documentation métier ou logicielle du projet (ex: commentaires, README) concernée.

## 3. Conformité aux Pratiques Collaboratives
- **Stratégie Git et Commits** : Antigravity est tenu de lire et appliquer à la lettre les guides suivants pour n'importe quelle démarche collaborative :
  - [`docs/GIT_STRATEGY.md`](docs/GIT_STRATEGY.md) (Règles de branches, interdiction de push direct sur main).
  - [`docs/COMMIT_CONVENTION.md`](docs/COMMIT_CONVENTION.md) (Formats spécifiques des commits et proscription formelle des commits "fourre-tout" inatomiques).
- **Directives IA et Garde-Fous spécifiques** : L'agent doit impérativement aligner son flux de travail sur ces documentations sous peine de voir son livrable refusé.
  - [`docs/AI_AGENT_RULES.md`](docs/AI_AGENT_RULES.md) (Modalités d'analyse, délimitation stricte, couverture).
  - [`docs/AI_GUARDRAILS.md`](docs/AI_GUARDRAILS.md) (Contrôle qualité ciblé, gestion des secrets, signalements d'absence de tests et de dettes techniques).
## 4. Traçabilité des Décisions et Intervention Humaine

La configuration impose de garder trace de ce qui l'IA décide :
- L'IA sollicite activement l'intervention de l'utilisateur pour valider ("Request User Approval") chaque phase risquée d'un projet.
- Elle génère systématiquement des journaux clairs pour les choix cornéliens, facilitant les revues ("Walkthrough" ou "Task" artifacts).
